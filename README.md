# 法務案件チェックポイント

https://h-kono-it.github.io/legal-check-helper

エンジニア向けの「逆引き」法務・経理リファレンスサイト。
[Blume](https://useblume.dev/) 製の静的サイトで、管理者が執筆した技術同人誌『Webエンジニアのための法務確認ガイド 〜「これ、法務案件では？」とアラートを出せるようになる本〜』を参考情報のベースにしている。

## コンセプト：逆引きファースト

法律の条文から学ぶのではなく、**「いま作ろうとしている機能」や「売ろうとしている商材」から、疑うべき法令へ引く**サイト。

すべてのページの起点は次の 2 つ：

1. **機能から引く**（`/features`、書籍 `80-appendix-checklist.md` 由来）
   「ポイント機能を作る」「DM 機能を作る」→ 疑うべき法令・確認事項へ
2. **商材から引く**（`/products`、書籍 `82-appendix-products.md` 由来）
   「酒を売る」「中古品を扱う」→ 必要な許認可・法令へ

### コンテンツ方針

- **1機能（1商材）= 1ページ**。書籍にない実務情報・追加要素を各ページに足していける構造にする（各ページ末尾の `{/* 実務メモ */}` 以下に追記）
- 書籍はあくまで参考情報。サイト独自のコンテンツを育てていく
- **章単位の「読み物」ページは持たない**。書籍の章の内容は各機能ページに吸収する。読ませたい深掘りが出てきたら、それは新しい機能ページ（または既存ページの実務メモ）にする

### 免責事項（必須要件）

法律を扱うサイトなので、**全ページに免責事項を表示する**：

- 必ず専門家の指示に従うこと
- このサイトの目的は法務的な判断を代替することではないこと
- 法律の解釈には揺らぎがあること
- 問題があっても一切責任はとらないこと

実装は `components/SiteFooter.astro`（全ページ共通フッター）+ `/disclaimer`（全文ページ）。レイアウトを変更するときもフッターの免責表示は消さないこと。

## サイト構成

```
docs/
├── index.mdx            # ハブ：2つの逆引きへの入口
├── disclaimer.mdx       # 免責事項の全文
├── 10-features/         # 逆引き①：機能から引く（/features、一覧表 + 1機能=1ページ）
├── 20-products/         # 逆引き②：商材から引く（/products、一覧表 + 1商材=1ページ）
└── 30-checklist/        # フェーズ別チェックリスト（/checklist、企画〜運用 + 法務相談準備）
```

フォルダ名の数字プレフィックスはサイドバーの並び順制御用で、URL には現れない（`10-features/` → `/features`）。

## 開発

```bash
npm install        # 済んでいれば不要
npm run dev        # 開発サーバー（ホットリロード）
npm run build      # 静的ビルド → dist/
npm run doctor     # 設定・コンテンツの診断
```

Node.js 22.12 以上が必要。
