# CLAUDE.md

## このプロジェクトは何か

エンジニア向け「逆引き」法務・経理リファレンスサイト。[Blume](https://useblume.dev/)（Astro + Vite ベースの静的サイトジェネレーター）で構築する。

このサイトは管理者が執筆した技術同人誌『Webエンジニアのための法務確認ガイド 〜「これ、法務案件では？」とアラートを出せるようになる本〜』の内容をもとに作成したもの。

[書籍頒布ページ](https://yomaigotolab.booth.pm/items/8344898)

## 最重要の設計原則

### 1. 逆引きファースト

すべてのコンテンツの起点は「機能から引く」（`/features`）と「商材から引く」（`/products`）の2つの一覧表。サイトは「作る機能」「売る商材」から疑うべき法令へ引ける構造にする。**章単位の「読み物」ページ（旧 `/chapters`）は置かない**（ブックサイト化を避けるため意図的に削除済み）。書籍の章にある深掘り内容は、該当する機能ページの本文に吸収する。新しいページを作るときは「逆引き表のどの行から到達するか」を必ず確認する。

### 2. 1機能（1商材）= 1ページ

逆引き表の各行は個別ページを持つ。書籍にない実務情報を追加しやすくするため。各ページ末尾の `{/* 実務メモ: ... */}` コメント以下が独自コンテンツの追記場所。

### 3. 全ページに免責事項（絶対要件）

法律を扱うサイトなので、全ページのフッターに免責事項を表示する。内容は「専門家の指示に従う」「法務判断の代替ではない」「法解釈には揺らぎがある」「一切責任を負わない」の4点。

- 実装: `components/SiteFooter.astro`（`components.ts` の `layout.Footer` スロットで全ページに注入）
- 全文: `docs/disclaimer.mdx`（`/disclaimer`）
- レイアウトやテーマを変更しても、この免責フッターを消してはならない

## ディレクトリ構成

- `docs/index.mdx` — ハブページ。逆引きへの入口
- `docs/10-features/` — 機能からの逆引き（→ `/features`）。`index.mdx` が一覧表、1機能=1ページ
- `docs/20-products/` — 商材からの逆引き（→ `/products`）
- `docs/30-checklist/` — フェーズ別チェックリスト（→ `/checklist`）
- 数字プレフィックスはサイドバー並び順の制御用。URL からは剥がれる（検証済み）
- 各フォルダの `meta.ts` がグループ名・ページ順を定義。ページ追加時は `meta.ts` の `pages` にも追記する
- `blume.config.ts` — サイト設定 / `components.ts` — レイアウトオーバーライド
- `BLUME.md` — Blume の使い方まとめ。**Blume の機能・設定に触るときはまずこれを読む**

## 開発コマンド

```bash
npm run dev      # 開発サーバー
npm run build    # 静的ビルド → dist/
npm run doctor   # 診断
```

依存は `npm install` 済み。Node.js 22.12 以上。

## Blume の要点（詳細は BLUME.md）

- `docs/` のファイルパス = URL。数字プレフィックス（`01-xxx.mdx`）で並び順制御、URL には出ない
- frontmatter: `title` / `description` / `draft` / `sidebar.{label,icon,badge,order}`
- frontmatterのtitleがh1で挿入されるため、本文のh1（#）は不要
- MDX の組み込みコンポーネント（`Card` / `CardGroup` / `Steps` / `Tabs` / `Accordion` など）は import 不要

## コンテンツ執筆ルール

- 言語は日本語
- **法令名・条文番号・数値**（例: 有効期限6ヶ月、残高1,000万円、60日以内支払い）は書籍ソースに忠実に。事実を創作しない
- 書籍にない独自コンテンツの追加は歓迎。ただし法的な記述を足すときは一次ソース（e-Gov・省庁ガイドライン）を確認し、断定を避け「専門家に確認」への導線を保つ
- 機能・商材ページの構成テンプレ: 冒頭に `| 項目 | 内容 |` テーブル（疑うべき法令・確認タイミング）→「なにが問題になるか」→「確認すべきこと」（`- [ ]`）→「こんな仕様が出たらアラート」→「あわせて確認」（関連する `/features/`・`/products/` ページへのリンク）→ `{/* 実務メモ */}`
- 書籍から内容を持ってくるときは、章単位でなく該当する機能ページに分解して吸収する。書籍内参照（「参照: 1章」等）はサイト内リンクに置き換える
- 書籍ソース側（`vivliostyle-legal-tech-book-2026`）のファイルは読み取り専用。編集しない
